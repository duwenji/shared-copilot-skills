# Kindle互換性チェックリスト

自動生成された EPUB/AZW3 形式の電子書籍が Kindle デバイスで正しく表示されるかを確認するためのチェックリストです。

---

## 📋 変換前のチェック

### マークダウンファイル構文
- [ ] **見出しレベルが正しい**
  - [ ] H1 は表紙とセクション開始のみ
  - [ ] H2, H3 が順階層になっている
  - [ ] H4以降が過剰に使用されていない

- [ ] **リスト形式が正なしい**
  - [ ] 箇条書き（`- `, `* `）が正しい
  - [ ] 番号付きリスト（`1. `, `2. `）が正しい
  - [ ] ネスト構造が正確

- [ ] **コードブロック**
  - [ ] バッククォート 3 つで囲まれている（` ``` `）
  - [ ] 言語指定がある（例: ` ```python `）
  - [ ] 改行が正しく保持されている

- [ ] **テーブル**
  - [ ] マークダウンテーブル形式が正しい
  - [ ] セルが多すぎない（Kindle は列幅が限定）

- [ ] **画像**
  - [ ] 存在しないパスが参照されていない
  - [ ] ファイルサイズが適正（< 1MB/画像）

- [ ] **リンク**
  - [ ] URL が正しい形式：`[テキスト](url)`
  - [ ] 相対パスが正確
  - [ ] 外部リンクが有効

---

## 📱 変換後の EPUB 検証

### ファイルサイズ
- [ ] 最終ファイルサイズが 100MB 以下
- [ ] 1 ファイルで複数ファイルが結合されている

### 目次（Table of Contents）
- [ ] 自動生成された目次が存在する
- [ ] 目次項目がすべての大見出しをカバーしている
- [ ] リンククリックで該当セクションに移動する

### スタイル・フォーマット
- [ ] テキストが左揃えまたは両端揃え
- [ ] 日本語フォントが使用可能
- [ ] 句点（。）の処理が正しい
- [ ] 見出しと本文の余白が適切

### ページ構造
- [ ] 各章がページ開始位置で始まっている（`page-break-before: always`）
- [ ] 見出しとテーブルが分断されていない
- [ ] コードブロックが完全に表示されている

### 特殊要素
- [ ] 脚注が正しくリンク化されている
- [ ] 引用ブロック（> ）が視覚的に区別される
- [ ] 水平線（`---` または `***`）が表示される

## ⚙️ よくある問題と解決策

### 文字化け・エンコード問題
```
問題: テキストが読めない文字になっている
❌ 原因: ファイルエンコードが UTF-8 でない
✅ 解決: すべてのマークダウンファイルを UTF-8 で保存
```

### テーブルが崩れている
```
問題: テーブルがはみ出したり、セルが重なっている
❌ 原因: 列が多すぎる、セル内容が長い
✅ 解決: テーブルを簡潔にするか、複数の小さなテーブルに分割
```

### コードが折り返されている
```
問題: コードが行の途中で切れている
❌ 原因: Kindle の表示幅より長いコード
✅ 解決: コード例を短縮するか、複数の小さなブロックに分割
```

### 目次が表示されない
```
問題: 目次ページがない、または不完全
❌ 原因: Pandoc の `--toc` オプション忘れ
✅ 解決: 変換スクリプトに `--toc` が含まれているか確認
```

### 日本語が正しく表示されない
```
問題: 日本語が漢字→ひらがなに変換されたり、崩れている
❌ 原因: フォント指定が不適切、メタデータに言語指定がない
✅ 解決: 
   1. metadata.yaml で language: ja を指定
   2. CSS で日本語フォントを allow-list に追加
```

### カバー画像が表示されない
```
問題: 書籍の表紙がない
❌ 原因: 画像ファイルが見つからない
✅ 解決: cover-image パスを確認、相対パスを修正
```

## 📤 Amazon Kindle Direct Publishing への申請

AZW3 をKindle ストアで販売する場合：

### 事前チェック
- [ ] ファイルが有効な AZW3 形式
- [ ] メタデータが確認
- [ ] 著作権情報が正確
- [ ] ページ数が妥当（通常 50ページ以上）

### 申請手順
1. [KDP コンソール](https://kdp.amazon.com) にログイン
2. 「新規タイトルを作成」
3. メタデータを入力
4. AZW3 ファイルをアップロード
5. 内容確認用 PDF を確認
6. 最終確認後に申請

### よくある却下理由
- [ ] テーブルが複雑で自動フォーマット失敗
- [ ] 画像の DPI が低い（72 DPI 以下）
- [ ] コンテンツが著作権侵害
- [ ] メタデータが不完全
